# Contributing

Thank you for your interest in contributing! The following guidelines help
maintainers review and accept changes quickly.

How to contribute

1. Fork the repository and create a feature branch: `git checkout -b feat/name`.
2. Make changes with small, focused commits.
3. Run any available tests and linters locally.
4. Open a pull request with a clear title and description.

Coding conventions

- Use 4-space indentation and follow PEP 8 for Python code.
- Add docstrings to modules, classes, and public functions.
- Write tests for new features and bug fixes.

Issue reporting

- Provide steps to reproduce, expected vs actual behavior, and platform
  information.

Pull request checklist

- [ ] Branch forks from `main` and targets `main`.
- [ ] Tests added/updated where applicable.
- [ ] Changelog entry included if public-facing behavior changed.
